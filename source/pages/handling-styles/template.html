Introduction

One aspect of web components I struggle with is that they never feel self-contained. The main place I feel that friction is with styles. Specifically:

  • I'm having to apply a display: block; in the parent page's CSS to get my components to render properly. If I want to make a component display inline I have to set that in the parent page's CSS as well. (It seems entirely plausible that I'm missing something here that would take care of this. It's certainly not obvious to me if that's true)
  • Components have access to the CSS variables set on the parent page. That's great as far as it goes, but the way I'd been thinking about it, it required the page that was using the component to have to adjust itself to accommodate whatever variable names the component expects

In both cases, using the component requires messing with the parent page's CSS. I really don't like the way that feels. I had a couple realizations last night about how to address those issues. This page is for taking a look at them.

Fair warning that I'm still new to web components so I may be off-base here. Part of the reason I'm doing this write up it to solicit feedback. With that said, here's what I'm doing.

Overview

There are two techniques I'm using to address working with styles:

  1. Add a stylesheet to the parent page from the component's .js file before creating the class that the component uses.
  2. Pass the names of CSS variables (or specific values) to use into the component via attributes.

Using those methods together I can use a component on a page without having to adjust the parent's CSS at all. The only thing I touch is the component tag itself.
